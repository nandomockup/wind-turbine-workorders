Wind Turbine Work Orders API Documentation

Overview

This API provides functionality to manage and retrieve work orders for wind turbine maintenance operations. The system enables efficient tracking and monitoring of maintenance activities across wind turbine installations.

Business Impact

  • Streamlines maintenance workflow management
  • Enables real-time tracking of work order status
  • Facilitates better resource allocation for maintenance teams
  • Improves operational efficiency through systematic work order management

API Endpoints

1. Get All Work Orders

Retrieves a list of all work orders in the system.

Endpoint: GET /api/WorkOrders

Response: Array of WorkOrder objects

[
  {
    "workOrderNumber": "WTWO-0001",
    "turbineId": "TURB-001",
    "description": "Maintenance for wind turbine 1",
    "status": "Closed"
  },
  {
    "workOrderNumber": "WTWO-0002",
    "turbineId": "TURB-002",
    "description": "Maintenance for wind turbine 2",
    "status": "Open"
  }
]

2. Get Work Order by Number

Retrieves a specific work order using its unique work order number.

Endpoint: GET /api/WorkOrders/{workOrderNumber}

Parameters: - workOrderNumber (string): Unique identifier for the work order (Format: WTWO-XXXX)

Response: Single WorkOrder object

{
  "workOrderNumber": "WTWO-0001",
  "turbineId": "TURB-001",
  "description": "Maintenance for wind turbine 1",
  "status": "Closed"
}

Data Model

WorkOrder

Field Type Description
WorkOrderNumber string Unique identifier (Format: WTWO-XXXX)
TurbineId string Associated turbine ID (Format: TURB-XXX)
Description string Description of maintenance work
Status string Current status ("Open" or "Closed")

Business Logic

  • Work orders are automatically generated for 15 sample turbines
  • Work order numbers follow the format WTWO-XXXX
  • Turbine IDs follow the format TURB-XXX
  • Status alternates between "Open" and "Closed" based on even/odd work order numbers
  • Returns 404 Not Found if requested work order doesn't exist
